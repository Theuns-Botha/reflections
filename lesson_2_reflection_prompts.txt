What happens when you initialize a repository? 
	- It initialises the hidden background directory used for tracking and manipulating the change metadata

Why do you need to do it?


	- So that you can start the revision control process

How is the staging area different from the working directory and the repository?

	- It is an intermediate area that can be used to control what changes you want to add with a commit

What value do you think it offers?
	- It helps you to break changes down into logical ones



How can you use the staging area to make sure you have one commit per logical
change?
	- By only adding files to the staging area tht corresponds to the logical change you want to commit


What are some situations when branches would be helpful in keeping your history 
organized?
	- Different logical progeressions can be tested with different branches while the main functional branch remains intact.

How would branches help?
	- You dont change your working source code when testing new unrelated features
